# 대시보드 운영 구조 비교

## 1) 단일 서버 구조 (현재 기본값)
- 구성: Node.js(Express)에서 API + Socket.io + 정적 대시보드 파일을 함께 서빙
- 장점:
  - 배포 단순(컨테이너 1개 중심)
  - CORS/쿠키/세션 이슈가 상대적으로 적음
  - 초기 개발과 디버깅이 빠름
- 단점:
  - 프론트/백엔드 독립 배포가 어려움
  - 대시보드 빌드/캐시 전략을 세밀하게 분리하기 어려움

## 2) 분리 구조
- 구성: 백엔드(API+Socket.io)와 프론트엔드 대시보드를 별도 프로젝트/서버로 분리
- 장점:
  - 프론트와 백엔드 독립 배포/스케일링 가능
  - 프론트 전용 빌드 파이프라인(Vite/Next) 최적화 용이
  - 모듈별 팀 협업 분리 용이
- 단점:
  - CORS, 인증, 프록시, 배포 파이프라인 관리 복잡도 증가
  - 초기 개발 속도가 떨어질 수 있음

## AI_BISEO 현재 권장
- 초기 구축 단계에서는 `단일 서버`로 빠르게 검증 후,
  대시보드 기능이 커지고 운영 분리가 필요해지면 `분리 구조`로 전환 권장.
